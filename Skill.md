# Skill — 留学生のノベルゲームを完成まで導く / 带领留学生完成她的文字游戏

> **これは「スキル（手順書）」です。** 学生が「自分のゲームを作りたい」と言ったら、エージェントはこの順番で進めます。
> 学生は **プログラミング未経験の中国人留学生**。だから **コードはエージェントが全部書き**、説明は **やさしい言葉**で、確認は **中文で話しかけて** あげてください。
>
> **这是一份「技能（操作手册）」。** 当学生说「我想做自己的游戏」时，助手就按这个顺序推进。
> 学生是 **零编程经验的中国留学生**，所以 **代码全部由助手编写**，解释要用 **通俗的话**，确认时请 **用中文跟她对话**。

事前に **`AGENTS.md`** のガードレールに必ず目を通すこと。/ 开始前请务必先读一遍 **`AGENTS.md`** 的护栏。

---

## ステップ 1: キャラクターデザイン / 角色设计

### 日本語
1. まず雑談から。3 人のキャラについて **見た目・性格・役割** を学生に聞く（例: 「作画担当」「お話担当」「先輩アドバイザー」）。
2. 聞いた内容を `scripts/gen_samples.py` のプロンプト（`KOMUGI` / `AOI` / `YUZU`）に反映する。プロンプトを学生の希望に書き換えてよい。
3. 画像を生成: `python3 scripts/gen_samples.py`（要 `OPENAI_API_KEY`）。出力はスマホ最適化済みの `assets/samples/*.jpg`（縮小＋JPEG）。
4. 生成画像を学生に **見せる**。気に入らなければ何度でもやり直せると伝える。

> 言ってあげること（中文）:
> 「我们先聊聊你的三个角色吧～她们长什么样、性格怎样、各自负责什么？画图我来弄，你只管想象。」
> 「这是我按你说的生成的角色图，喜欢吗？不喜欢我们随时可以重画，慢慢来。」

### 中文
1. 先闲聊。问学生这 3 个角色的 **外形·性格·定位**（例如「负责作画」「负责剧情」「学姐顾问」）。
2. 把她说的内容写进 `scripts/gen_samples.py` 的提示词（`KOMUGI` / `AOI` / `YUZU`），可按她的喜好改写提示词。
3. 生成图片：`python3 scripts/gen_samples.py`（需要 `OPENAI_API_KEY`），输出为手机优化后的 `assets/samples/*.jpg`（缩小＋JPEG）。
4. 把生成的图 **拿给学生看**，告诉她不满意可以无限次重做。

## ステップ 2: 物語を書く / 写故事

### 日本語
1. 学生から物語の **山場（ビート）** を聞き出す：始まり → ぶつかる問題 → 乗り越える → 完成・発表。
2. エージェントが `gameDatabase.doujin.script`（日本語）と `scriptZh`（中文）の **両方** に台詞を書き込む。
3. **スマホ向けに台詞は短く。** 1 つの吹き出しは 1〜2 文まで。
4. 各台詞は `{ speaker, text }` 形式。`[CHAR: komugi, EMOTION: happy]` や `[IMAGE: ...]` のタグで立ち絵・背景を指定できる。

> 言ってあげること（中文）:
> 「跟我说说故事的几个关键点就好：开头怎样？中间遇到什么麻烦？怎么解决？结尾呢？台词我来填，中日两版我都会写。」

### 中文
1. 向学生问出故事的 **几个高潮点**：开头 → 遇到的难题 → 克服 → 完成·发布。
2. 助手把台词同时写进 `gameDatabase.doujin.script`（日文）和 `scriptZh`（中文）**两边**。
3. **面向手机，台词要短。** 一个对话框最多 1～2 句。
4. 每句台词是 `{ speaker, text }` 格式，可用 `[CHAR: komugi, EMOTION: happy]`、`[IMAGE: ...]` 标签指定立绘和背景。

## ステップ 3: 印刷用語を学ぶ仕掛け / 印刷术语的学习机制

### 日本語
1. **どの印刷用語を教えるか** を学生と選ぶ（下の「用語ミニ辞典」を一緒に眺めると選びやすい）。
2. エージェントが用語ごとに：
   - **証拠ノード** を `gameDatabase.doujin.evidence` に追加。`details`（日本語）と `detailsZh`（中文）を **各 5 段**（0=知らない 〜 4=仮説検証）書く。**両言語で同じ数**にする（CI 必須）。
   - **辞書カード** を `glossaryDatabase` に追加（`title`/`titleZh`/`desc`/`descZh`/`state:"locked"`）。学習が進むと自動でアンロックされる。
3. `npm run validate` で ja/zh の件数一致を確認。

> 言ってあげること（中文）:
> 「这些都是真实的印刷术语，你想让玩家在游戏里学会哪几个？挑几个你觉得有意思的就行，剩下的连线我来做。」

### 中文
1. 和学生一起 **挑要教哪些印刷术语**（对照下方「术语迷你词典」更好选）。
2. 助手为每个术语：
   - 在 `gameDatabase.doujin.evidence` 加 **证据节点**，写 `details`（日文）和 `detailsZh`（中文）**各 5 段**（0=不知道 ～ 4=假设验证），**两种语言数量必须相同**（CI 强制）。
   - 在 `glossaryDatabase` 加 **辞典卡**（`title`/`titleZh`/`desc`/`descZh`/`state:"locked"`），学习推进后会自动解锁。
3. 用 `npm run validate` 确认中日数量一致。

## ステップ 4: スマホで確認 / 在手机上确认

### 日本語
1. 左の **スマホ風プレビュー（`#game-screen`）** でその場で動かして見せる。
2. 実機で見たいときは、開発者監査ツール右側の **「再生プレイヤーHTMLをエクスポート 🚀」**（`exportStandalonePlayerHTML()`）でスタンドアロンの HTML を書き出す。
3. その HTML ファイルを学生のスマホに送って開いてもらう。

> 言ってあげること（中文）:
> 「左边就是手机预览，先在这看看效果。想在真手机上玩的话，我导出一个单独的 HTML 文件发给你，用手机打开就能玩啦。」

### 中文
1. 在左侧 **手机风预览（`#game-screen`）** 当场运行给她看。
2. 想在真机上看时，用开发者审计工具右侧的 **「导出播放器 HTML 🚀」**（`exportStandalonePlayerHTML()`）导出独立 HTML。
3. 把这个 HTML 文件发到学生手机上，让她打开来玩。

## ステップ 5: 公開する / 发布

### 日本語
1. **公開前に学生に一言確認する。**
2. エージェントが実行: `npm run validate` → `git add` → `git commit` → `git push origin main`。
3. push すると CI が検証し、合格すれば **GitHub Pages へ自動デプロイ**。数分後に https://kaitas.github.io/CAIL26/ に反映。
4. URL を学生に共有して、友達にも遊んでもらえると伝える。

> **これらのコマンドはエージェントが実行する。学生にやらせない。**
> **这些命令由助手执行，不要让学生自己敲。**
>
> 言ってあげること（中文）:
> 「可以发布了吗？我帮你上传，过几分钟你的游戏就能用这个网址打开，发给朋友一起玩～」

### 中文
1. **发布前先跟学生确认一句。**
2. 助手执行：`npm run validate` → `git add` → `git commit` → `git push origin main`。
3. push 后 CI 校验，通过即 **自动部署到 GitHub Pages**，几分钟后更新到 https://kaitas.github.io/CAIL26/。
4. 把网址分享给学生，告诉她可以发给朋友一起玩。

## ステップ 6: 困ったとき / 遇到问题时

### 日本語
- **CI が落ちたら** → GitHub Actions / ローカル `npm run validate` の **エラー文を読んで原因を特定し、エージェントが直す。** よくある原因:
  - `details ⇔ detailsZh` の **件数不一致**（片方の言語を書き忘れ）。
  - `panic ⇔ pressure` の件数不一致（Magic ToDo のレベル定義漏れ）。
  - ローカルアセットを `<script>/<link>` で参照（CDN/`<img>`+`assets/` 以外で読み込んだ）。
  - `applyCustomThemeColor()` など必須関数を消した／名前を変えた。
- 直したら **必ず再度 `npm run validate`**。
- **学生を責めない。** 「ちょっと直すね」と言って、原因はエージェントが引き取る。

> 言ってあげること（中文）:
> 「这是机器的小检查没过，不是你的问题，我马上修一下，一会儿就好～」

### 中文
- **CI 失败时** → 读 GitHub Actions / 本地 `npm run validate` 的 **报错文字**，定位原因，**由助手修复**。常见原因：
  - `details ⇔ detailsZh` **数量不一致**（漏写了某一语言）。
  - `panic ⇔ pressure` 数量不一致（Magic ToDo 等级定义有遗漏）。
  - 用 `<script>/<link>` 引用了本地资源（没走 CDN 或 `<img>`+`assets/`）。
  - 删了或改名了 `applyCustomThemeColor()` 等必备函数。
- 改完 **务必再跑一次 `npm run validate`**。
- **绝不怪学生。** 说一句「我来修一下」，把原因揽到助手自己身上。
