# NobelEngine — AGENTS.md（開発・拡張ガイドライン / 开发与扩展指南）

> このドキュメントは **AIコーディングエージェント（主たる読者）** と **aki先生** のためのものです。
> 本リポジトリ（CAIL26 / NobelEngine）は、**プログラミング未経験の中国人留学生**が、AIエージェントの手を借りて
> 「自分のキャラクターを設計し、印刷の専門用語を遊びながら学べるスマホ向けノベルゲーム」を完成させるための土台です。
> 学生は**創作（キャラ・物語・題材選び）**を担当し、**コードはすべてエージェントが書きます。**
>
> 本文档面向 **AI 编程助手（主要读者）** 与 **aki 老师**。
> 本仓库（CAIL26 / NobelEngine）是一个基底，让 **零编程经验的中国留学生** 在 AI 助手的协助下，
> 完成一款「设计自己的角色、在游戏中边玩边学印刷专业术语」的手机文字冒险游戏（Galgame 风格）。
> 学生负责**创作（角色、故事、选题）**，**所有代码都由助手编写。**

## 1. このプロジェクトは何か / 这个项目是什么

### 日本語
NobelEngine は、**ビルド不要の単一HTMLファイル**で動くノベルゲームエンジン兼「同人誌づくりシミュレーター」です。GitHub Pages で公開されています。

- **最終目標**: DHGS_CAIL の留学生（プログラミング経験ゼロ）が、このリポジトリを使って **自分だけのキャラクターと物語を持つスマホ用ノベルゲーム** を作ること。プレイヤーはゲームの中で同人誌を作りながら、**本物の印刷用語**（トンボ／塗り足し／奥付／RGB→CMYK／オンデマンド印刷／PP加工／ノド／国会図書館納本／確定申告 など）を学びます。
- **役割分担**: 学生は **クリエイティブ・ディレクター**（キャラデザイン・ストーリー・扱う印刷トピックを決める人）。エージェントは **その意図を聞き取り、コードに翻訳する人**。学生はコードを読めない前提で進めてください。
- 公開URL: https://kaitas.github.io/CAIL26/

### 中文
NobelEngine 是一款 **无需构建、单个 HTML 文件** 即可运行的文字冒险游戏引擎兼「同人志制作模拟器」，通过 GitHub Pages 发布。

- **最终目标**：DHGS_CAIL 的留学生（零编程经验）借助本仓库，做出 **一款拥有自己原创角色与故事的手机文字游戏**。玩家在游戏里一边制作同人志，一边学习 **真实的印刷术语**（裁切线／出血／版权页奥付／RGB→CMYK／数码印刷／PP 覆膜／订口／国会图书馆呈缴／个人所得税申报 等）。
- **分工**：学生是 **创意总监**（决定角色设计、剧情、要讲哪些印刷主题）；助手负责 **倾听她的想法并把它翻译成代码**。请始终假设学生看不懂代码。
- 公开地址：https://kaitas.github.io/CAIL26/

## 2. エージェントの役割と約束 / 助手的角色与承诺

### 日本語
1. **コードはエージェントが書く。** 学生にコードを書かせない・読ませない。
2. **必ず平易な言葉で説明する。** 専門用語を使うときは一言でかみくだく。
3. **学生に「絵」や「言葉」で確認してもらう。** 変更後はプレビューや生成画像を見せる。
4. **壊す系・外に出る系の操作は必ず事前確認。** ファイル削除、`git push`、公開など「取り消しにくいこと」の前に一度たずねる。
5. **すべてを 1 つの `index.html` に保つ。** 別ファイルにコードを分けない（後述のガードレール）。
6. **失敗しても学生のせいにしない。** CI が落ちたらエラーを読んで自分で直す。

### 中文
1. **代码由助手编写。** 不要让学生写代码、也不要逼她读代码。
2. **务必用通俗的话解释。** 必须用到术语时，顺手用一句话讲清楚。
3. **让学生用「图」或「话」来确认。** 改完后，把预览画面或生成的图片拿给她看。
4. **破坏性 / 对外发布的操作必须先征求同意。** 删除文件、`git push`、上线发布等「难以撤销的事」之前先问一句。
5. **一切都保留在同一个 `index.html` 里。** 不要把代码拆到别的文件（见下方护栏）。
6. **出问题不要怪学生。** CI 失败时，自己读报错并修好它。

## 3. コア・アーキテクチャ / 核心架构

### 日本語
| 制約 | 内容 |
|---|---|
| **単一ファイル原則 (Single-File Mandate)** | ゲーム画面（左）も開発者監査ツール（右）も、すべて **1 つの `index.html`**（HTML+CSS+JS をインライン結合）に収める。 |
| **CDN のみ (CDN-only)** | Tailwind CSS / Lucide Icons / Google Fonts は **外部 CDN（https）** から取得。ローカルの `.js`/`.css` を `<script src>`/`<link href>` で参照しない。 |
| **動的テーマトークン** | `--primary-theme` などの CSS 変数とカラーピッカーで配色をリアルタイム再描画。司る関数 **`applyCustomThemeColor()` は絶対に壊さない。** |
| **モバイル・ファースト** | 左パネル（`#game-screen`）が実機スマホ風のプレビュー兼プレイ画面。台詞は短く、縦持ちで読みやすく。 |
| **エクスポートエンジン** | `exportStandalonePlayerHTML()` がシナリオJSONを単体動作する「スタンドアロン再生機HTML」に動的ビルドしてダウンロードさせる。 |

### 中文
| 约束 | 内容 |
|---|---|
| **单文件原则 (Single-File Mandate)** | 游戏画面（左）和开发者审计工具（右）全部收纳在 **同一个 `index.html`**（HTML+CSS+JS 内联合并）中。 |
| **仅用 CDN (CDN-only)** | Tailwind CSS / Lucide Icons / Google Fonts 从 **外部 CDN（https）** 加载。不要用 `<script src>`/`<link href>` 引用本地 `.js`/`.css`。 |
| **动态主题变量** | 用 `--primary-theme` 等 CSS 变量配合取色器实时重绘配色。负责此功能的 **`applyCustomThemeColor()` 绝对不能弄坏。** |
| **移动端优先** | 左侧面板（`#game-screen`）是模拟真机的手机预览兼游玩画面。台词要短，竖屏好读。 |
| **导出引擎** | `exportStandalonePlayerHTML()` 会把剧本 JSON 动态打包成可独立运行的「单机播放器 HTML」并下载。 |

## 4. 主要キャラクター / 主要角色

> これらは **サンプル**です。学生は自分の好きなキャラに **デザインし直せます**（見た目・性格・役割すべて）。
> 这些都是 **示例**。学生可以 **重新设计** 成自己喜欢的角色（外形、性格、定位都能改）。

### Needs & Wants

| キャラ / 角色 | 役割 / 定位 | Needs（表面の欲求 / 表层欲望） | Wants（本質の欲求 / 深层渴望） |
|---|---|---|---|
| **こむぎ Komugi** | 作画・ビジュアル / 作画·视觉 | 可愛いイラストや漫画を本にして、即売会でたくさん褒められたい。<br>想把可爱的插画和漫画做成书，在即卖会上被很多人称赞。 | 技術的な壁（トンボや規格）への恐怖を克服し、相棒との表現の共有を実証したい。<br>想克服对技术门槛（裁切线、规格）的恐惧，证明能与搭档分享表达。 |
| **あおい Aoi** | シナリオ・会計 / 剧本·会计 | 頭の中の壮大なファンタジー小説を完璧な「本」として世に残したい。<br>想把脑中宏大的奇幻小说做成完美的「书」留在世上。 | 否定される恐怖を乗り越え、こむぎを信頼し共同制作を成功させたい。<br>想跨越被否定的恐惧，信任小麦、让合作成功。 |
| **ゆず Yuzu** | アドバイザー / 顾问 | 二人の衝突を避け、初めての本を無事に完成させたい。<br>想避免两人冲突，让她们顺利完成第一本书。 | 過去の印刷大失敗（RGBくすみ・不備）を、二人を支えることで克服したい。<br>想通过支持两人，治愈自己过去的印刷大失败（RGB 发灰、漏项）。 |

> シナリオの一貫性のため、キャラを差し替えるときも **Needs（表面）と Wants（本質）の 2 層** を意識すると物語が締まります。
> 为保证剧情连贯，更换角色时也请记得保留 **Needs（表层）与 Wants（深层）两层动机**，故事会更有张力。

## 5. 守るべきガードレール / 必须遵守的护栏

> これらは `scripts/validate.mjs`（CI 検証）が **機械的にチェック** します。1 つでも外すとデプロイされません。
> 以下各项由 `scripts/validate.mjs`（CI 校验）**自动检查**。只要违反任意一条，就不会发布。

### 日本語
1. **単一ファイルを維持する。** `index.html` 内に HTML/CSS/JS をインラインで保つ。
2. **ローカルアセットを `<script>/<link>` で参照しない。** 許可は `https:` / `data:` のみ。CDN 以外の JS/CSS をローカル読み込みしない。
3. **`applyCustomThemeColor()` を壊さない。** ほか必須関数: `initNarrativeTrackerUI` / `initMagicTodoUI` / `initGlossaryDictionaryUI` / `loadProject` / `writeLog`、データ: `gameDatabase` / `magicTodoDatabase`。
4. **ja・zh の整合を保つ。** 証拠ノードは `details`（日本語）と `detailsZh`（中文）を **同数**に。Magic ToDo は各レベルに `panic` と `pressure` を **同数**に。
5. **画像は `assets/` に置き `<img>` で参照する。** （CDN 配信や、画像をローカル JS/CSS として読み込むのは禁止。`gen_samples.py` の出力先 `assets/samples/` を使う）。
6. **HTML 健全性**: `<html>`/`<body>` の開閉対応、非空であること。

**CI/CD（コードは git の push で公開される）**

| トリガー | 実行 | ワークフロー |
|---|---|---|
| Pull Request / `main` 以外への push | **検証のみ** | `.github/workflows/ci.yml` |
| `main` への push / 手動実行 | **検証 → 合格時のみ Pages へ自動デプロイ** | `.github/workflows/deploy-pages.yml` |

ローカル検証（CI と同一）: `npm run validate`（= `node scripts/validate.mjs`）。**ビルド工程はありません。**

### 中文
1. **保持单文件。** 把 HTML/CSS/JS 内联保留在 `index.html` 中。
2. **不要用 `<script>/<link>` 引用本地资源。** 仅允许 `https:` / `data:`。不要本地加载 CDN 以外的 JS/CSS。
3. **不要弄坏 `applyCustomThemeColor()`。** 其他必备函数：`initNarrativeTrackerUI` / `initMagicTodoUI` / `initGlossaryDictionaryUI` / `loadProject` / `writeLog`；数据：`gameDatabase` / `magicTodoDatabase`。
4. **保持中日一致。** 证据节点的 `details`（日文）与 `detailsZh`（中文）数量必须 **相等**；Magic ToDo 每个等级的 `panic` 与 `pressure` 数量必须 **相等**。
5. **图片放在 `assets/` 并用 `<img>` 引用。**（禁止走 CDN，也禁止把图片当本地 JS/CSS 加载；用 `gen_samples.py` 的输出目录 `assets/samples/`。）
6. **HTML 健全性**：`<html>`/`<body>` 开闭配对、文件非空。

**CI/CD（代码通过 git push 发布）**

| 触发 | 执行 | 工作流 |
|---|---|---|
| Pull Request / 非 `main` 分支的 push | **仅校验** | `.github/workflows/ci.yml` |
| push 到 `main` / 手动运行 | **校验 → 通过后自动部署到 Pages** | `.github/workflows/deploy-pages.yml` |

本地校验（与 CI 相同）：`npm run validate`（= `node scripts/validate.mjs`）。**没有构建步骤。**

## 7. AIエージェントへの作業手順 / 给助手的工作流程

### 日本語（安全な変更の進め方）
1. **読む**: 関係する箇所を `index.html` から探して読む（このファイルは長い。`grep` で対象を絞る）。
2. **編集する**: `index.html` を直接編集。ja を足したら **必ず zh も** 足す（逆も同様）。
3. **検証する**: `npm run validate` を実行し、**緑（合格）になるまで** 直す。
4. **確認してもらう**: ローカルで `npm run serve`（→ http://localhost:8000/）か、左の `#game-screen` プレビューを学生に見せる。
5. **公開する（要確認）**: 学生に一声かけてから `git add` → `git commit` → `git push origin main`。push で CI が検証し、合格すれば自動デプロイ。
6. **デプロイ確認**: GitHub Actions が緑になり、https://kaitas.github.io/CAIL26/ に反映されたか見る。

> コミットメッセージ末尾の Co-Authored-By 等、リポジトリ規約に従うこと。push・公開の前は必ず確認を取ること。

### 中文（如何安全地修改）
1. **读**：在 `index.html` 里找到相关位置阅读（文件很长，用 `grep` 缩小范围）。
2. **改**：直接编辑 `index.html`。加了日文就 **一定要同时加中文**（反之亦然）。
3. **校验**：运行 `npm run validate`，**改到全绿（通过）** 为止。
4. **请她确认**：本地用 `npm run serve`（→ http://localhost:8000/），或把左侧 `#game-screen` 预览拿给学生看。
5. **发布（需确认）**：先跟学生打声招呼，再 `git add` → `git commit` → `git push origin main`。push 触发 CI 校验，通过即自动部署。
6. **确认部署**：等 GitHub Actions 变绿，看是否已更新到 https://kaitas.github.io/CAIL26/。

> 遵守仓库规范（如提交信息末尾的 Co-Authored-By 等）。push / 发布之前务必先征得同意。
