NobelEngine - 開発・拡張ガイドライン (AGENTS.md)

本ドキュメントは、NobelEngine（クラシック判事＆同人誌製作・ナラティブ設計スイート）を改変・拡張する共同開発者およびAIエージェントに向けた技術設計書、および開発ロードマップです。

1. プロジェクト概要

NobelEngineは、プレイヤーの「知らない（Unknown）」という認知状態を、ゲーム内の「探求」や「検証」といったステップを経て「到達（Resolution）」へ深化させる体験設計（Narrativeデザイン）を検証するためのシミュレーター兼ゲームエンジンです。

中学生が直面する同人誌製作のリアルな諸問題（原稿トンボ、予算計算、奥付義務、国会図書館納本、税務）を学び、体験の深度に比例してToDoタスクが爆発的に急増する「タスク雪崩（Task Avalanche）」といった現実の創作の恐怖と面白さを電子的に可視化します。

2. コア・アーキテクチャ

本システムは以下の制約のもとで強固に設計されています。

シングルファイル・マイルストーン (Single-File Mandate):

原則として、全てのゲームプレイルーム（左側）と、開発者監査ツール（右側）は単一の index.html 内に収められています。

Tailwind CSS および Lucide Icons を外部CDNより取得し、追加のnpmインストールやビルド環境を構築することなく即時実行可能です。

デザイン・トークンの動的制御:

--primary-theme, --primary-light, --bg-theme-main などのCSS Variables（カスタムプロパティ）を使用し、リアルタイムカラーピッカーを介して全体のカラースキーム（フォント、コントラスト、境界線）を動的に再描画します。

エクスポートエンジン:

index.html 内で編集・コンパイルされたシナリオJSON配列を自動でパースし、ブラウザで直接動作する別ファイルの「スタンドアロン再生機（HTMLファイル）」を動的にビルドしてダウンロードさせることができます。

3. 主要エージェント（キャラクター）設計

シナリオの一貫性を担保するため、登場人物の内発的・外発的動機（Needs & Wants）を以下のようにコード及び設定上定義しています。

キャラクター

役割

Needs（表面的な欲求）

Wants（本質的な欲求）

こむぎ (Komugi)

作画・ビジュアル

可愛いイラストや漫画を本にして、即売会でたくさんの人に褒められたい。

技術的な壁（トンボや規格）への恐怖を克服し、相棒との表現の共有を実証したい。

あおい (Aoi)

シナリオ・会計

頭の中の壮大なファンタジー小説を完璧な「本」としてこの世に残したい。

他者に否定される恐怖を乗り越え、こむぎを信頼し、共同制作を成功させたい。

ゆず (Yuzu)

アドバイザー

二人の衝突を回避し、初めての本を無事に完成させられるようサポートしたい。

過去の自分の印刷大失敗（RGBくすみ、不備）を、二人を支えることで克服したい。

4. 特殊機能モジュールと改変ルール

① ナラティブ・フラグ・トラッカー (initNarrativeTrackerUI())

挙動: 証拠品や課題をクリックするごとに level が 0 (知らない) から 4 (仮説検証/Evidence) まで上昇。

拡張方法: 新たな情報を追加する場合は、gameDatabase オブジェクト内の evidence に一意のキーと details 配列（0〜4のテキスト）を多言語分記述してください。

② Magic ToDo & Task Avalanche (initMagicTodoUI())

挙動: goblin.toolsオマージュ。シンプルな目標が「細分化」されると、3段階の解像度ツリーが展開されます。最高レベルに達すると想定時間が爆発（20時間 ➔ 1200時間）し、心理集中度ゲージが「不安（Anxiety）」にシフトします。

拡張方法: 新たなToDoバーストを設計するには、magicTodoDatabase に新たな levels（panic, pressure, tasks 配列）を構造化して定義してください。

③ 獲得型製本用語辞書 (initGlossaryDictionaryUI())

挙動: 原稿サイズや予算計算のナラティブレベルが上がると、対応する専門用語（トンボ、オンデマンド、ノド、PP加工、RGB-CMYK変換、奥付）の辞書カードが「未理解 🔒」から「理解済 ✨」に変化し、アンロックされます。

5. エージェント開発ロードマップ (今後の改造プラン)

本プロジェクトを今後拡張していく場合、以下のマイルストーンに沿って実装することを推奨します。

マイルストーン 1: シナリオタグパーサーの高度化

現状: エクスポートされたプレイヤーは [IMAGE: ...] や [CHAR: ...] などの基本タグを簡易背景色変化で代替しています。

改善: スタンドアロンプレイヤー内で、CSSグラフィックやアニメーションアセットを用いた完全なビジュアル演出をサポートするローカル描画コードの強化。

マイルストーン 2: 選択肢分岐分岐ツリーの視覚化

改善: 「Script設計」タブ内に、シーンの接続関係をグラフとして視覚化するマップ表示（SVGやCanvasベース）を追加。ドラッグ＆ドロップでストーリーを並び替え可能にする。

マイルストーン 3: 印刷所シミュレータの実装

改善: ページ数、背幅、用紙の坪量（kg）、部数、PP加工を決定することで、リアルタイムに見積価格が連動・変化し、頒布価格決定時に「損益分岐点（トントン以上）」をダイナミックに試算する会計ツールを完全に可視化する。

マイルストーン 4: クラウドセーブ/コラボレーション (Firestore対応)

改善: 複数ユーザーが同じシナリオを共同で書き換え、リアルタイムに双方のプレビューに反映させるマルチプレイヤーエディタの実装。

6. AIエージェントへの約束事 (System Prompt Guardrail)

コードをリファクタリング、あるいは新規タグ機能を追加する際は、必ず 単一ファイル（HTML, CSS, JSがインライン結合された状態） を維持すること。

カラーテーマを動的反映するための applyCustomThemeColor() を破壊しないようにすること。

多言語対応（ja / zh）の翻訳JSONを辞書やToDoテキストを含めて整合させること。
